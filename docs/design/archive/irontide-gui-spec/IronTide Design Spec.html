IronTide GUI — Design Specification

A full-featured BitTorrent client for engineers and power users, written in Rust with a Slint-rendered GUI. Feature parity with qBittorrent across all menus, settings, and workflows, delivered in an original visual language with three skins, three layout variants, and five novel capabilities.

1 · Design principles

Power-user first

Every feature qBittorrent has must be reachable without hunting. Defaults are sane for new users; every knob is exposed for advanced ones.

Data density with air

Balanced density by default (32px rows). Compact and spacious are one click away. No information is truncated when it wouldn't be in qBittorrent.

Keyboard-driven

Every action has a shortcut. ⌘K opens a command palette that routes to any torrent, setting, or action.

Original, not derivative

We do not mimic qBittorrent's chrome, iconography, or color palette. Feature parity ≠ UI parity.

Slint-honest

Designs respect Slint's native widget set; custom components are called out explicitly with implementation notes.

Quiet by default

The UI stays out of the way during heavy transfers. Status is communicated through color and the sparkline in the status bar, not modals.

4 · Layout variants

The prototype offers three layout models, user-switchable at runtime and per-workspace-persistent. All three share the same toolbar, sidebar, and status bar.

L1 · 3-pane (default)

Sidebar | torrent list (top) | details pane (bottom). The classic qBittorrent/Deluge layout. Best for single-monitor users who want everything in view.

L2 · Inspector drawer

Sidebar | torrent list (full height) | 460px right drawer. Fits more rows; drawer collapses to gain space.

L3 · Command workspace

Minimal chrome. Sidebar can collapse to icons or hide entirely. Details shown on-demand via Enter or ⌘I. Suited to large torrent counts and keyboard users.

6 · Per-torrent details

Six tabs, all in the prototype:

  • General — transfer stats (progress, speeds, ratio, ETA, availability, pieces), information block (hash v1/v2, save path, dates, comment, privacy flag), error card when applicable, and a 48×8 piece-availability heatmap.
  • Trackers — DHT/PeX/LSD pseudo-trackers shown first; user trackers below with status, peer/seed/leech counts, last-announce, message.
  • Peers — IP/port, connection type (BT/µ), protocol flags with legend, client/version, progress bar, rates, relevance.
  • HTTP Sources — web seeds (BEP-19).
  • Content — recursive file tree with per-file progress and priority (High/Normal/Low/Skip). Toolbar: Sequential, First/last pieces first.
  • Speed — 60s/5m/1h/1d speed graph, plus per-torrent rate overrides.

7 · Add torrent flow

Modal dialog with three source tabs: File, Magnet, URL. All tabs share the same preview card (name, size, pieces, file count, comment, hash) and options panel:

  • Save path (with quick-browse)
  • Category + Tags
  • Start paused / Skip hash check / Pre-allocate / Sequential / First-last / Alt limits / Auto-manage
  • Per-file selection with priority dropdown

The magnet flow supports multiple magnets (one per line, batched). URL flow accepts http/https and is fetched in-app.

8 · Torrent creation tool

Standalone page (also reachable as a modal). Source file/folder, tracker list (blank line = tier break), web seeds, piece size (Auto / 16 KiB → 16 MiB), format (v1 / v2 / Hybrid BEP-52), privacy flag, optimize alignment, source tag, comment. Shows hashing-time estimate live.

9 · Preferences

Eight tabs. Every setting from qBittorrent's preferences is present; where names differ, the qBittorrent equivalent is noted in the implementation doc (not shown in UI).

Behavior

Theme, Skin, Density, Language, deletion-confirm, pause-all confirm, double-click action, shortcut set (Default/Emacs/Vim/qBittorrent parity). Startup: login-start, start-minimized, minimize-to-tray-on-close, resume-previous-session. Notifications: on-complete, on-error, on-RSS-match, sound, external program (with %N %F %D %I substitution).

Downloads

Default save path, separate incomplete folder, .!it extension, auto-categories, add-torrent dialog, start-paused, skip-hash, pre-allocate, append-date, smart category suggestion. Watched folder, .torrent copy/move paths, delete-on-add. On-complete: move-to folder, external program.

Connection

Incoming port, random-on-start, UPnP/NAT-PMP, port status badge. Connection limits (global + per-torrent + upload slots + active dl/ul/total). Proxy: None / HTTP / SOCKS4/5, hostname lookups via proxy. IP filter enable + path + refresh-on-startup.

Speed

Global DL/UL limits with toggles. Alternative limits + scheduled hours (preset dropdown + custom). Transport-overhead / µTP / LAN-exempt toggles.

BitTorrent

DHT, PeX, LSD, encryption (Prefer/Require/Disable), anonymous mode. Seeding limits (ratio / seed time / inactive time → Pause / Remove / Remove+data / Super-seed). Queueing toggle + slow thresholds.

RSS

Enable, refresh interval, max articles. Auto-download on/off, smart episode filter, smart-filter regex, repack handling.

Web UI

Enable, bind address + port, HTTPS (cert/key paths). Auth: username, password, bypass-on-localhost, session timeout, login-attempts ban. Security: clickjacking/CSRF/host-header, allowed hosts, reverse-proxy X-Forwarded-For trust list. Dynamic DNS (DynDNS / No-IP / DuckDNS / Cloudflare).

Advanced

libtorrent tuning (async-I/O threads, hashing threads, file-pool size, outstanding-memory-checking, disk cache, cache expiry, coalesce R/W, piece-extent affinity, socket buffers). Network: interface selection, bind IP, reported-IP mode, update check, resolve peer countries/hostnames. Behavior: strict super-seeding, announce-tier rules, µTP-TCP mixed mode, always-announce, resume-data interval. Export config (reveal config.toml), reset to defaults.

10 · RSS reader + auto-download

Two-pane: feeds sidebar (with unread counts + last-updated) + items list + a rules docked panel at the bottom. Each rule: enable toggle, name, must-contain, must-not-contain, episode filter, smart filter, category, save path, feeds it applies to.

11 · Search plugins

Query box + plugin filter + category filter. Results table: name, size, seeds, peers, engine, date, Add button (which routes to the Add-Torrent dialog pre-filled with the URL). Plugin manager sidebar: enable/disable per plugin, install from URL, update all.

12 · Bandwidth scheduler

7×24 grid (days × hours). Each cell is one of three states: Full speed · Alternative limits · Paused. Click to toggle, drag to paint. Presets: Work hours, Overnight, Weekends. Current active state banner below.

13 · IP filter

Stats strip (enabled, total ranges, blocked today, last refresh) + editable range table (range, level = block/allow, comment). Banned-peers-this-session list with reasons. Import from URL, .p2p/.dat format support.

14 · Logs & events

Monospace log view with level filters (INFO/WARN/ERROR/DEBUG), tail mode, clear, export. Rows: timestamp (ms), level pill, message. Auto-scroll pauses on user interaction.

15 · Statistics

10 stat cards (all-time DL/UL/ratio/shared-time, session DL/UL/uptime, global peers, DHT nodes, active/max connections). 90-day rolling transfer graph. Top peer countries bar chart. Cache & I/O table (read-cache hit rate, write queue, disk R/W latencies, piece DL time).

16 · Web UI remote access

Dedicated tool page shows live endpoint info (local/LAN/external URLs, status, uptime, active sessions). Active sessions table with revoke button. Pair-a-device panel with QR (URL + user + one-time token). All config settings live in Preferences → Web UI.

17 · Categories, tags, save-path rules

Categories are single-valued and drive default save paths. Tags are multi-valued and free-form. Both show in the sidebar with counts and drive the filter. Save-path rules support token expansion: {category}, {tracker}, {yyyy}/{mm}, {content_type}.

18 · Queue management

Queueing is toggleable in Preferences → BitTorrent. Caps: max active DL, max active UL, max active total. Slow-torrent exemption (configurable KB/s thresholds). Sidebar chevrons on selected rows move queue position up/down; ⌘↑/⌘↓.

20 · Novel features new

1 · Command palette (⌘K)

Every action, navigation target, setting, and torrent is reachable by typing. Fuzzy-matched with category headers. Keyboard-only users never need the mouse.

2 · Smart category suggestion

On add-torrent, a local classifier (no network) inspects file names and sizes to suggest a category and save path. Routes "ubuntu-*.iso" to Linux, "*.epub" to Books, etc. User-trainable via right-click → "Train as: X" on any existing torrent.

3 · Bandwidth intent, not limits

Instead of (only) hard KB/s caps, users can set goals: "Leave 10 Mbps free for video calls," "Never exceed 80% of measured uplink," "Pause during Zoom/Meet." IronTide probes bandwidth every few minutes and auto-adjusts.

4 · Pair-to-phone QR

In Web UI → Pair a device, a QR encodes URL + username + one-time token. IronTide Mobile scans it and is authed permanently. Tokens rotate and revoke from the active-sessions table.

5 · Verify-before-download

When a torrent's files already exist in the save path (common when re-adding after reinstall), IronTide offers a fast local hash-verify pass before downloading anything. If all pieces match, it goes straight to seeding. Saves massive amounts of traffic on re-adds.

Rust-side boundaries

The UI layer is pure Slint; data comes from a Rust state object exposed via slint::Model implementations:

  • TorrentModel → backs the torrent table. Updates diff'd against the previous snapshot, emitted via model_changed on the affected row only.
  • PeerModel, TrackerModel, FileTreeModel → backed by the currently-selected torrent. Swap wholesale on selection change.
  • LogModel → ring buffer, capped at ~10k entries. Level filter applied Slint-side via a FilterModel.
  • Stats, SpeedSample → simple in-out property on the root, polled by a timer at 1 Hz.

Callouts

Virtualization. With 1,000+ torrents, ListView virtualization is mandatory. Keep row delegates lightweight — no per-row sparklines; use a single off-screen sparkline component reused for the selected row only.
Theme switching at runtime. Slint globals support re-assignment, but we recommend updating every token from Rust in one atomic batch to avoid flicker during skin changes.
Accessibility. Slint's accessible-* properties are supported since 1.4. Tag every interactive element — especially the torrent rows (accessible-role: list-item, accessible-label) — during implementation.

Handoff

The prototype at IronTide Prototype.html is the source of truth for layout, spacing, and interactions. Toggle Tweaks in the toolbar to see all skins, densities, and layout variants. Hover over any row, tab, or button to see full hover/focus states.
