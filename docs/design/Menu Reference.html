IronTide — Menu Reference
Every application-menu item rendered open at once — the static source of truth for the menu bar. This is the same model that drives components/chrome.jsx (MENU_MODEL) and is documented as tables in DESIGN.md §1a. Build these as custom styled popups in Slint, not the native MenuBar widget — see the note at the bottom.

Geometry. Menu bar 28px tall, --bg-1. Dropdown panel: --bg-2 fill, 1px --border-1, radius --r-md (6px), --shadow-lg, 5px padding. Each item row 27px tall; 16px leading gutter reserved for the check/radio mark; label Inter 12.5px --fg-0; accelerator JetBrains Mono 11px --fg-3, right-aligned; submenu chevron --fg-2. Hover row = --bg-hover. Separator = 1px --divider with 4px vertical margin.

State marks. Toggles (Details Panel, Row Striping, Emerald Glow) show an emerald check when on. Radio groups (Sidebar, Density) show one emerald dot on the active option. These bind to the live Tokens/tweak state, not static values.

Slint implementation — important. Slint's built-in MenuBar { Menu { MenuItem } } delegates to the platform's native menu renderer, which ignores our color tokens — it will look like a stock OS menu, not this design. To get the Breeze/emerald look shown here, build a custom menu: a MenuButton-style row per top-level label that opens a PopupWindow containing a styled VerticalLayout of item rows (a reusable MenuItemRow component for label + accelerator + check/radio/chevron). Submenus = a nested PopupWindow opened on hover/right-edge. Drive it all from one data model mirroring MENU_MODEL. Register the accelerators in accel.rs independently so shortcuts work whether or not the menu is open.
